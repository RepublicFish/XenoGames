# XenoGames 需求文档

## 项目概述

XenoGames 是一个个人单用户游戏记录应用，用于记录游戏通关感受、评分及截图。用户可以管理自己的游戏库，记录游玩过程中的操作感受、剧情感悟，并对已通关的游戏进行评分。

## 核心目标

- 建立一个私人的游戏通关记录库
- 记录游玩过程中的感受（操作手感、游戏机制等）
- 记录通关后的感悟（剧情、结局评价等）
- 对已通关游戏进行评分
- 通过截图留存游戏精彩瞬间

## 功能需求

### F1 - 游戏管理

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F1.1 | 新增游戏 | 填写游戏信息（名称、平台、状态等）入库 | P0 |
| F1.2 | 编辑游戏 | 修改已录入的游戏信息 | P0 |
| F1.3 | 删除游戏 | 删除游戏及其关联的截图、标签关系 | P0 |
| F1.4 | 游戏列表 | 以卡片网格形式展示所有游戏，支持分页 | P0 |
| F1.5 | 游戏详情 | 查看单个游戏的完整信息、感受、截图 | P0 |

### F2 - 搜索与筛选

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F2.1 | 关键词搜索 | 按游戏名称模糊搜索 | P0 |
| F2.2 | 平台筛选 | 按平台（PC/PS5/Switch 等）过滤 | P0 |
| F2.3 | 状态筛选 | 按游玩状态（PLAYING/COMPLETED/ABANDONED）过滤 | P0 |
| F2.4 | 标签筛选 | 按标签（RPG/FPS 等）过滤游戏 | P1 |

### F3 - 标签系统

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F3.1 | 新增标签 | 创建自定义标签（名称 + 颜色） | P1 |
| F3.2 | 标签列表 | 查看所有已有标签 | P1 |
| F3.3 | 删除标签 | 删除不再使用的标签 | P1 |
| F3.4 | 关联游戏 | 在新增/编辑游戏时可选择多个标签 | P1 |

### F4 - 截图管理

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F4.1 | 上传截图 | 在游戏详情中上传截图，支持批量上传 | P1 |
| F4.2 | 浏览截图 | 在游戏详情中以网格/瀑布流查看截图 | P1 |
| F4.3 | 删除截图 | 删除单张截图 | P2 |
| F4.4 | 截图排序 | 拖拽调整截图展示顺序 | P2 |

### F5 - 图片上传

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F5.1 | 封面上传 | 在游戏表单中上传封面图片 | P0 |
| F5.2 | 截图上�� | 在游戏详情中上传截图 | P1 |

## 非功能需求

| 编号 | 类型 | 描述 |
|------|------|------|
| NF1 | 性能 | 游戏列表分页加载，单页不超过 20 条 |
| NF2 | 可用性 | 响应式布局，支持 PC 端主流分辨率 |
| NF3 | 数据安全 | 单用户本地部署，无需登录鉴权 |
| NF4 | 可维护性 | 前后端分离，接口遵循 RESTful 规范 |

## 用户故事

- **US1**: 作为一个玩家，我希望能记录我玩过的每一款游戏，包括游戏名称、平台和游玩状态，以便追踪我的游戏历程。
- **US2**: 作为一个玩家，我希望能写下游戏过程中的操作感受和通关后的剧情感悟，以便日后回顾。
- **US3**: 作为一个玩家，我希望能给通关的游戏打分（1-10），以便对比不同游戏的体验。
- **US4**: 作为一个玩家，我希望能给游戏打上标签（如 RPG、FPS），以便按类型浏览我的游戏库。
- **US5**: 作为一个玩家，我希望能上传游戏截图，留住精彩瞬间。
- **US6**: 作为一个玩家，我希望能通过搜索和筛选快速找到某款游戏。
